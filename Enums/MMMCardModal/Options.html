Options

Options for presenting the MMMCardModal.

Set custom options by modifying self.options after initializing. If you want to edit the options after the init call, you should override it with a new MMMCardModal.Options object.

  • AnimationConstant
    See more
  • AnimationType
    See more
  • SnapshotPolicy
    See more
  • isScaleEffectEnabled

    If the presenting viewController (snapshot thereof) should be scaled out (iOS 13+ style). Defaults to true.

  • isFadeEffectEnabled

    If the presenting viewController (snapshot thereof) should be faded out. Defaults to true.

  • topPadding

    The amount of space from the top of the screen to the modal.

  • backgroundColor

    The background color behind the snapshot of the presenting viewController.

  • cardCornerRadius

    The cornerRadius of the card where the presented viewController lives.

  • cardBackgroundColor

    The backgroundColor of the card where the presented viewController lives.

  • dragVelocity

    The amount of velocity it takes to stick to the next StickPosition.

  • closeVelocity

    The amount of velocity it takes to close.

  • captureScrollViewVelocityDivision

    The amount to divide the velocity by when it comes from a captured scrollview, UIScrollViews have a tendency to pass really high velocities. Defaults to 2 (e.g. velocity will be divided by 2).

  • dragResistance

    The resistance when the card is dragged up too far.

  • animationType

    The policy for updating the constraints, if your view has constraints set to the bottom layout guide this can cause unwanted behaviour. By default we don’t update the top constraint, the view is only transformed.

    Options: - transform; Only transform the view. - constraints; Only animate the top constraint, don’t transform.

  • animationDuration

    The duration of the complete animation, in seconds. Defaults to 0.5 seconds.

  • animationDamping

    The damping applied to the animation spring, in seconds. Defaults to 0.7.

  • animationVelocity

    The velocity of to the animation spring. Defaults to 0.7.

  • animationAlpha

    The maximum amount the presenting viewController will be faded out, only used when isFadeEffectEnabled is true.

  • animationScale

    The maximum amount the presenting viewController will be scaled, only used when isScaleEffectEnabled is true.

  • animationTransform

    The maximum amount the presenting viewController will be transformed, so it’s still visible, only used when isScaleEffectEnabled is true.

  • animationTransformToSafeArea

    If the animationTransform should respect the safeArea.

  • animationCornerRadius

    The maximum amount of the cornerRadius applied to the presenting viewController, only used when isScaleEffectEnabled is true.

  • animateAfterOrientationChange

    If the modal should animate back to it’s stickPosition after an orientation change, or if this should happen directly.

  • alternativeSnapshotView

    Alternative view to take the snapshot from, e.g. for when the presenting viewController is not full screen.

  • snapshotPolicy

    How we should take snapshots. See documeentation in SnapshotPolicy for more info on each case.

  • init()

    Initialize with default values.

  • init(block:)

    Initialize new Options object with a callback to set the required options.

    Usage:

    .init {
    $0.backgroundColor = .red
    $0.cardCornerRadius = 25
}
